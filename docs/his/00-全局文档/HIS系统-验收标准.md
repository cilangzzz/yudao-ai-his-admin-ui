# YUDAO-AI-HIS 智慧医疗信息系统 - 验收标准

> **文档编号**: YUDAO-HIS-AC-001
> **版本**: V1.0
> **编制日期**: 2026-06-15
> **编制方法**: 采用Gherkin格式（Given-When-Then），覆盖正常流程、异常流程、边界条件
> **关联文档**: YUDAO-HIS-PRD-001, YUDAO-HIS-US-001

---

## 1. 文档说明

### 1.1 验收标准格式

采用BDD（Behavior-Driven Development）的Gherkin语法：
- **Given**: 前置条件
- **When**: 用户行为/触发动作
- **Then**: 预期结果
- **And**: 附加条件

### 1.2 覆盖原则

每个功能覆盖三类场景：
- **正常流程**: 标准业务操作
- **异常流程**: 错误处理、异常情况
- **边界条件**: 极端值、边界情况

## 2. M09 系统管理验收标准

### AC-SYS-001: 用户账号管理

```gherkin
Feature: 用户账号管理

  Scenario: 创建用户账号成功
    Given 系统管理员已登录系统
    And 管理员拥有"用户管理"权限
    When 管理员进入"系统管理 > 用户管理"页面
    And 点击"新建用户"按钮
    And 填写用户名 "doctor001"、姓名 "张医生"、工号 "D001"、科室 "内科"、角色 "门诊医生"
    And 点击"保存"
    Then 系统创建用户账号成功
    And 自动生成初始密码
    And 用户 "doctor001" 可以使用初始密码登录系统

  Scenario: 用户名重复
    Given 系统中已存在用户名 "doctor001"
    When 管理员尝试创建同名用户
    Then 系统提示 "用户名已存在"
    And 用户创建失败

  Scenario: 必填字段为空
    Given 管理员在新建用户页面
    When 管理员未填写用户名就点击保存
    Then 系统提示 "用户名为必填项"
    And 用户创建失败

  Scenario: 停用用户账号
    Given 系统中存在用户 "doctor001"
    When 管理员将该用户状态改为"停用"
    Then 用户 "doctor001" 无法登录系统
    And 系统提示 "账号已停用，请联系管理员"

  Scenario Outline: 用户名边界值验证
    Given 管理员在新建用户页面
    When 管理员输入用户名 "<username>"
    Then 验证结果为 "<result>"

    Examples:
      | username   | result                     |
      | ab         | 无效（少于3位）            |
      | abc        | 有效                       |
      | a          | 无效（少于3位）            |
      | admin@#    | 无效（含特殊字符）         |
      | abcdefghij12345678901 | 无效（超过20位） |
```

### AC-SYS-002: 角色权限管理

```gherkin
Feature: 角色权限管理 (RBAC)

  Scenario: 创建角色并分配权限
    Given 管理员在角色管理页面
    When 创建角色 "门诊医生"
    And 分配菜单权限：门诊管理、门诊医生工作站
    And 分配按钮权限：新增、修改、查看
    And 分配数据权限：本科室
    Then 角色创建成功
    And 分配该角色的用户登录后只能看到授权菜单
    And 只能操作授权按钮
    And 只能查看本科室数据

  Scenario: 权限变更实时生效
    Given 用户 "doctor001" 已登录系统
    And 该用户拥有"处方开立"权限
    When 管理员撤销该用户的"处方开立"权限
    And 用户刷新页面
    Then 用户无法看到"处方开立"功能

  Scenario: 越权操作拦截
    Given 用户 "nurse001" 角色为"护士"
    And 该角色无"处方开立"权限
    When 用户尝试直接访问处方开立URL
    Then 系统返回403错误
    And 显示 "无权访问该功能"
```

    And CDS校验通过
    And 处方开立成功

  Scenario: 开立处方 - 药物相互作用警告
    Given 医生正在开立处方
    And 患者正在服用 "华法林"
    When 医生添加药品 "阿司匹林"

    And 医生可选择"继续开方"或"修改处方"

  Scenario: 开立处方 - 剂量合理性提示
    Given 医生为80岁老年患者开立处方
    When 医生设置药品剂量超出老年人推荐剂量
    Then 系统CDS弹出提示："患者年龄80岁，建议调整剂量为推荐范围"

  Scenario: ICD-10诊断编码选择
    Given 医生在诊断开立界面
    When 输入关键字 "高血压"
    Then 系统显示ICD-10编码列表：
      | 编码    | 名称                    |
      | I10     | 原发性高血压            |
      | I11     | 高血压性心脏病          |
      | I12     | 高血压性肾脏病          |
