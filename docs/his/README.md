## 3. 前端开发工作流

### 3.1 开发流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                    HIS 前端开发工作流                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐  │
│  │ 阅读文档  │───→│ 生成API  │───→│ 创建页面  │───→│ 联调测试  │  │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘  │
│       │              │              │              │           │
│       ▼              ▼              ▼              ▼           │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐  │
│  │ 理解业务  │    │ 定义类型  │    │ 实现组件  │    │ 验证功能  │  │
│  │ 规则流程  │    │ 封装请求  │    │ 路由配置  │    │ 提交代码  │  │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 每个子模块开发步骤

1. **Step 1: 阅读理解文档**
   - 阅读 `功能点需求.md` 理解业务需求
   - 阅读 `API接口.md` 理解接口定义
   - 阅读 `业务规则.md` 理解业务约束

2. **Step 2: 生成 API 层**
   - 在 `apps/web-antd/src/api/his/` 下创建接口文件
   - 定义 TypeScript 类型接口
   - 封装 API 请求方法

3. **Step 3: 创建页面组件**
   - 在 `apps/web-antd/src/views/his/` 下创建页面
   - 实现列表、表单、详情等组件
   - 配置路由和菜单

4. **Step 4: 联调测试**
   - 对接后端 API
   - 测试业务流程
   - 修复 Bug
