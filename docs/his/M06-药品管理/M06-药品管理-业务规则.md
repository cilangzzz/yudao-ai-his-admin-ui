#### BR-PHARM-001: 药品入库验收规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PHARM-001 |
| 规则名称 | 药品入库验收规则 |
| 规则类型 | 校验规则 |
| 适用对象 | 药品入库(drug_inbound) |
| 规则描述 | 药品入库必须验收品名、规格、批号、效期、数量，验收合格方可入库 |
| 触发时机 | 药品入库操作时 |
| 校验逻辑 | ```
// 伪代码
FUNCTION validateInbound(inboundDTO):
    // 1. 基础校验
    IF inboundDTO.drugId IS NULL THEN
        RETURN Error("药品ID不能为空")
    IF inboundDTO.batchNo IS NULL OR inboundDTO.batchNo = '' THEN
        RETURN Error("批号为必填项")
    IF inboundDTO.expireDate IS NULL THEN
        RETURN Error("效期为必填项")
    IF inboundDTO.quantity <= 0 THEN
        RETURN Error("入库数量必须大于0")
    
    // 2. 效期校验
    minExpiryMonths = getConfig('minExpiryMonths', 6)
    monthsToExpiry = DATEDIFF(inboundDTO.expireDate, NOW()) / 30
    IF monthsToExpiry < minExpiryMonths THEN
        needConfirm = true
        warning = "药品效期不足" + minExpiryMonths + "个月，是否继续入库？"
    
    // 3. 批号重复校验
    existingStock = SELECT * FROM drug_stock 
                    WHERE drug_id = inboundDTO.drugId 
                    AND batch_no = inboundDTO.batchNo
    IF existingStock EXISTS THEN
        RETURN Prompt("该批号已存在，是否合并库存？")
    
    // 4. 采购订单关联校验
    IF inboundDTO.orderId IS NOT NULL THEN
        order = SELECT * FROM purchase_order WHERE id = inboundDTO.orderId
        IF order.status != 'APPROVED' THEN
            RETURN Error("采购订单未审批通过")
    
    RETURN Success
``` |
| 失败处理 | 返回具体错误信息，阻止入库操作 |
| 优先级 | P0 |

#### BR-PHARM-002: 药品出库库存校验规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PHARM-002 |
| 规则名称 | 药品出库库存校验规则 |
| 规则类型 | 校验规则 |
| 适用对象 | 药品出库(drug_outbound) |
| 规则描述 | 出库数量必须小于等于当前可用库存，库存不足时拒绝出库 |
| 触发时机 | 药品出库操作时 |
| 校验逻辑 | ```
// 伪代码
FUNCTION validateOutbound(outboundDTO):
    // 1. 基础校验
    IF outboundDTO.quantity <= 0 THEN
        RETURN Error("出库数量必须大于0")
    
    // 2. 库存校验
    availableStock = SELECT SUM(quantity) FROM drug_stock 
                     WHERE drug_id = outboundDTO.drugId
                     AND status IN ('NORMAL', 'NEAR_EXPIRY')
                     AND warehouse_type = outboundDTO.fromWarehouseType
    
    IF availableStock < outboundDTO.quantity THEN
        RETURN Error("库存不足，当前库存" + availableStock + 
                    getUnit(outboundDTO.drugId) + "，申请数量" + 
                    outboundDTO.quantity + getUnit(outboundDTO.drugId))
    
    // 3. 批号库存校验（指定批号时）
    IF outboundDTO.batchNo IS NOT NULL THEN
        batchStock = SELECT quantity FROM drug_stock 
                     WHERE drug_id = outboundDTO.drugId
                     AND batch_no = outboundDTO.batchNo
        IF batchStock < outboundDTO.quantity THEN
            RETURN Error("指定批号库存不足")
    
    RETURN Success
``` |
| 失败处理 | 返回库存不足提示，拒绝出库操作 |
| 优先级 | P0 |

#### BR-PHARM-003: CDS药物相互作用检查规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PHARM-003 |
| 规则名称 | CDS药物相互作用检查规则 |
| 规则类型 | 校验规则 |
| 适用对象 | 处方审核(prescription_review)、医嘱审核(order_review) |
| 规则描述 | 处方开立时必须检查药物-药物相互作用，按风险等级分级警告 |
| 触发时机 | 处方/医嘱提交时、处方审核时 |
| 校验逻辑 | ```
// 伪代码
FUNCTION checkDrugInteraction(drugList):
    interactions = []
    
    // 两两组合检查
    FOR i = 0 TO drugList.length - 1:
        FOR j = i + 1 TO drugList.length - 1:
            drug1 = drugList[i]
            drug2 = drugList[j]
            
            // 查询相互作用知识库
            interaction = SELECT * FROM drug_interaction_knowledge
                         WHERE (drug_id_a = drug1.id AND drug_id_b = drug2.id)
                            OR (drug_id_a = drug2.id AND drug_id_b = drug1.id)
            
            IF interaction EXISTS THEN
                // 风险等级判定
                riskLevel = interaction.risk_level  // HIGH/MEDIUM/LOW
                result = {
                    drug1: drug1.name,
                    drug2: drug2.name,
                    riskLevel: riskLevel,
                    mechanism: interaction.mechanism,
                    clinicalEffect: interaction.clinical_effect,
                    suggestion: interaction.suggestion
                }
                interactions.add(result)
    
    // 按风险等级排序
    interactions.sortBy(riskLevel, DESC)  // HIGH > MEDIUM > LOW
    
    RETURN interactions

// 风险等级定义
// HIGH: 禁止配伍，必须更换替代药物
// MEDIUM: 需密切监测，有替代方案建议  
// LOW: 可配伍，需注意监测
``` |
| 失败处理 | 高风险：强制退回处方；中风险：需确认处理；低风险：显示提示 |
| 优先级 | P0 |

#### BR-PHARM-004: CDS过敏检查规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PHARM-004 |
| 规则名称 | CDS过敏检查规则 |
| 规则类型 | 校验规则 |
| 适用对象 | 处方审核(prescription_review)、医嘱审核(order_review) |
| 规则描述 | 基于患者过敏史检查药物过敏风险，包括直接过敏、交叉过敏、成分过敏 |
| 触发时机 | 处方/医嘱提交时、处方审核时 |
| 校验逻辑 | ```
// 伪代码
FUNCTION checkAllergy(patientId, drugList):
    allergyWarnings = []
    
    // 获取患者过敏史
    patientAllergies = SELECT * FROM patient_allergy WHERE patient_id = patientId
    
    IF patientAllergies IS EMPTY THEN
        RETURN { status: 'DATA_MISSING', message: '患者过敏史缺失，请确认患者无药物过敏' }
    
    // 遍历处方药品
    FOR drug IN drugList:
        // 1. 直接过敏检查
        FOR allergy IN patientAllergies:
            drugCategory = getDrugCategory(drug.id)
            IF allergy.allergen_category = drugCategory THEN
                allergyWarnings.add({
                    type: 'DIRECT_ALLERGY',
                    drugName: drug.name,
                    allergen: allergy.allergen,
                    riskLevel: 'HIGH',
                    suggestion: '禁止使用，更换替代药物'
                })
        
        // 2. 交叉过敏检查
        crossAllergies = SELECT * FROM drug_cross_allergy
                         WHERE drug_id = drug.id
        FOR crossAllergy IN crossAllergies:
            FOR allergy IN patientAllergies:
                IF crossAllergy.related_allergen = allergy.allergen THEN
                    allergyWarnings.add({
                        type: 'CROSS_ALLERGY',
                        drugName: drug.name,
                        allergen: allergy.allergen,
                        riskLevel: 'MEDIUM',
                        crossRate: crossAllergy.cross_rate,
                        suggestion: '谨慎使用，建议皮试后用药'
                    })
        
        // 3. 成分过敏检查
        drugIngredients = getDrugIngredients(drug.id)
        FOR ingredient IN drugIngredients:
            FOR allergy IN patientAllergies:
                IF hasAllergenStructure(ingredient, allergy.allergen) THEN
                    allergyWarnings.add({
                        type: 'INGREDIENT_ALLERGY',
                        drugName: drug.name,
                        ingredient: ingredient.name,
                        riskLevel: 'MEDIUM',
                        suggestion: '药品含过敏相关成分，谨慎使用'
                    })
    
    // 按风险等级排序
    allergyWarnings.sortBy(riskLevel, DESC)
    
    RETURN allergyWarnings
``` |
| 失败处理 | 高风险过敏：强制退回处方；中风险：需确认处理 |
| 优先级 | P0 |

#### BR-PHARM-005: CDS剂量合理性检查规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PHARM-005 |
| 规则名称 | CDS剂量合理性检查规则 |
| 规则类型 | 校验规则 |
| 适用对象 | 处方审核(prescription_review)、医嘱审核(order_review) |
| 规则描述 | 基于患者年龄、体重、肾功能、肝功能检查剂量合理性 |
| 触发时机 | 处方/医嘱提交时、处方审核时 |
| 校验逻辑 | ```
// 伪代码
FUNCTION checkDosage(patientId, drug):
    dosageWarnings = []
    
    // 获取药品剂量知识
    dosageKnowledge = SELECT * FROM drug_dosage_knowledge WHERE drug_id = drug.id
    
    // 获取患者信息
    patient = SELECT * FROM patient WHERE id = patientId
    age = calculateAge(patient.birthDate)
    weight = patient.weight
    gfr = patient.gfr  // 肾小球滤过率
    
    // 1. 常规剂量检查
    normalMin = dosageKnowledge.adult_min_dose
    normalMax = dosageKnowledge.adult_max_dose
    prescribedDose = drug.dose * drug.frequency  // 日剂量
    
    IF prescribedDose > normalMax THEN
        ratio = prescribedDose / normalMax
        riskLevel = ratio > 2.0 ? 'HIGH' : 'MEDIUM'
        dosageWarnings.add({
            type: 'DOSE_TOO_HIGH',
            prescribed: prescribedDose,
            max: normalMax,
            ratio: ratio,
            riskLevel: riskLevel,
            clinicalRisk: dosageKnowledge.overdose_risk,
            suggestion: '请确认剂量，建议减量至' + normalMin + '-' + normalMax
        })
    ELSE IF prescribedDose < normalMin THEN
        dosageWarnings.add({
            type: 'DOSE_TOO_LOW',
            prescribed: prescribedDose,
            min: normalMin,
            riskLevel: 'LOW',
            suggestion: '剂量低于常规最低剂量，可能影响疗效'
        })
    
    // 2. 老年人剂量调整
    IF age >= 65 AND dosageKnowledge.elderly_adjust = true THEN
        elderlyDose = dosageKnowledge.elderly_dose
        IF prescribedDose > elderlyDose THEN
            dosageWarnings.add({
                type: 'ELDERLY_ADJUST',
                riskLevel: 'MEDIUM',
                message: '老年患者肾脏清除率降低',
                recommended: elderlyDose,
                suggestion: '老年人推荐剂量：' + elderlyDose
            })
    
    // 3. 儿童剂量计算
    IF age < 18 AND dosageKnowledge.pediatric_formula IS NOT NULL THEN
        childDose = calculateChildDose(dosageKnowledge.pediatric_formula, weight, age)
        dosageWarnings.add({
            type: 'PEDIATRIC_DOSE',
            riskLevel: 'INFO',
            recommended: childDose,
            formula: dosageKnowledge.pediatric_formula
        })
    
    // 4. 肾功能剂量调整
    IF dosageKnowledge.renal_adjust = true THEN
        IF gfr IS NULL THEN
            dosageWarnings.add({
                type: 'DATA_MISSING',
                riskLevel: 'LOW',
                message: '患者肾功能数据缺失，无法进行肾功能剂量调整检查'
            })
        ELSE IF gfr < 60 THEN
            renalDose = getRenalAdjustedDose(dosageKnowledge, gfr)
            dosageWarnings.add({
                type: 'RENAL_ADJUST',
                riskLevel: 'MEDIUM',
                gfr: gfr,
                gfrLevel: getGFRLevel(gfr),
                recommended: renalDose,
                suggestion: 'GFR ' + gfr + 'ml/min：' + renalDose.suggestion
            })
    
    // 5. 体重相关剂量计算
    IF dosageKnowledge.weight_based = true AND weight IS NOT NULL THEN
        weightDose = dosageKnowledge.weight_dose_min + '-' + dosageKnowledge.weight_dose_max
        calculatedDose = weightDose * weight
        dosageWarnings.add({
            type: 'WEIGHT_BASED',
            riskLevel: 'INFO',
            perKg: weightDose + 'mg/kg',
            calculated: calculatedDose + 'mg'
        })
    
    RETURN dosageWarnings
``` |
| 失败处理 | 高风险：强制退回；中风险：需确认；低风险/提示：显示建议 |
| 优先级 | P0 |

#### BR-PHARM-006: CDS配伍禁忌检查规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PHARM-006 |
| 规则名称 | CDS配伍禁忌检查规则 |
| 规则类型 | 校验规则 |
| 适用对象 | 处方审核(prescription_review)、医嘱审核(order_review) |
| 规则描述 | 检查静脉用药配伍禁忌，包括物理性、化学性配伍禁忌 |
| 触发时机 | 处方/医嘱提交时、处方审核时 |
| 校验逻辑 | ```
// 伪代码
FUNCTION checkIncompatibility(drugList):
    incompatibilities = []
    
    // 筛选静脉用药
    ivDrugs = drugList.filter(drug -> drug.route = 'IV' OR drug.route = 'IV_DRIP')
    
    IF ivDrugs.length < 2 THEN
        RETURN { status: 'SKIP', message: '非静脉用药或单药，无需检查' }
    
    // 两两组合检查
    FOR i = 0 TO ivDrugs.length - 1:
        FOR j = i + 1 TO ivDrugs.length - 1:
            drug1 = ivDrugs[i]
            drug2 = ivDrugs[j]
            
            // 查询配伍禁忌知识库
            incompat = SELECT * FROM drug_incompatibility_knowledge
                       WHERE (drug_id_a = drug1.id AND drug_id_b = drug2.id)
                          OR (drug_id_a = drug2.id AND drug_id_b = drug1.id)
            
            IF incompat EXISTS THEN
                incompatibilities.add({
                    drug1: drug1.name,
                    drug2: drug2.name,
                    type: incompat.incompatibility_type,  // PHYSICAL/CHEMICAL
                    riskLevel: incompat.risk_level,
                    description: incompat.description,
                    consequence: incompat.consequence,
                    suggestion: incompat.suggestion
                })
    
    // 3. 溶媒配伍检查
    FOR drug IN ivDrugs:
        IF drug.solvent IS NOT NULL THEN
            solventCompat = SELECT * FROM drug_solvent_compatibility
                           WHERE drug_id = drug.id AND solvent_type = drug.solvent
            IF solventCompat EXISTS AND solventCompat.is_compatible = false THEN
                incompatibilities.add({
                    type: 'SOLVENT_INCOMPATIBLE',
                    drug: drug.name,
                    solvent: drug.solvent,
                    riskLevel: 'MEDIUM',
                    suggestion: solventCompat.suggestion
                })
    
    RETURN incompatibilities
``` |

// 伪代码
FUNCTION validateNarcoticPrescriptionAuth(doctorId, drugId):
    // 1. 检查药品是否为麻醉药品
    drug = SELECT * FROM drug WHERE id = drugId
    IF drug.drug_type != 'NARCOTIC' THEN
        RETURN Success  // 非麻醉药品无需校验
    
    // 2. 检查医生处方权
    doctorAuth = SELECT * FROM doctor_prescription_auth
                 WHERE doctor_id = doctorId
                 AND auth_type = 'NARCOTIC'
                 AND status = 'ACTIVE'
                 AND expire_date >= NOW()
    
    IF doctorAuth IS NULL THEN
        // 记录违规操作
        INSERT INTO security_log (type, doctor_id, action, result, time)
        VALUES ('NARCOTIC_AUTH_FAIL', doctorId, 'PRESCRIBE_NARCOTIC', 'REJECTED', NOW())
        
        RETURN Error("您不具有麻醉药品处方权")
    
    // 3. 检查处方权限级别
    narcoticLevel = drug.narcotic_level  // 麻醉药品级别
    IF doctorAuth.auth_level < narcoticLevel THEN
        RETURN Error("您的处方权级别不足，该药品需更高级别处方权")
    
    RETURN Success
``` |

#### BR-PHARM-008: 麻醉药品双人开柜验证规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PHARM-008 |
| 规则名称 | 麻醉药品双人开柜验证规则 |
| 规则类型 | 校验规则 |
| 适用对象 | 麻醉药品专柜(narcotic_cabinet) |
| 规则描述 | 开启麻醉药品专柜必须双人双锁验证，两位不同授权人员刷卡确认 |
| 触发时机 | 开启麻醉药品专柜时 |
| 校验逻辑 | ```
// 伪代码
FUNCTION validateDualPersonOpen(cabinetId, person1Id, person2Id):
    // 1. 检查两人是否为同一人
    IF person1Id = person2Id THEN
        INSERT INTO security_log (type, person_id, action, result, time)
        VALUES('DUAL_VERIFY_SAME_PERSON', person1Id, 'OPEN_CABINET', 'REJECTED', NOW())
        RETURN Error("验证人员重复，需要两位不同授权人员")
    
    // 2. 检查两人是否都有麻醉药品管理权限
    FOR personId IN [person1Id, person2Id]:
        auth = SELECT * FROM narcotic_admin_auth
               WHERE person_id = personId
               AND status = 'ACTIVE'
        IF auth IS NULL THEN
            INSERT INTO security_log (type, person_id, action, result, time)
            VALUES('UNAUTHORIZED_ACCESS', personId, 'OPEN_CABINET', 'REJECTED', NOW())
            RETURN Error("人员" + personId + "无麻醉药品管理权限")
    
    // 3. 记录开柜操作
    INSERT INTO cabinet_open_log (cabinet_id, person1_id, person2_id, open_time, status)
    VALUES (cabinetId, person1Id, person2Id, NOW(), 'OPENED')
    
    // 4. 解锁专柜
    UPDATE narcotic_cabinet SET status = 'UNLOCKED', last_open_time = NOW()
    WHERE id = cabinetId
    
    // 5. 设置超时锁定任务（5分钟后）
    scheduleTask('CABINET_TIMEOUT_LOCK', cabinetId, NOW() + 5 MINUTES)
    
    RETURN Success
``` |
| 失败处理 | 返回"需要双人双锁验证"或"无麻醉药品管理权限"，拒绝开柜 |
| 优先级 | P0 |

#### BR-PHARM-009: 处方审核状态校验规则

// 伪代码
FUNCTION validatePrescriptionForDispense(prescriptionId):
    prescription = SELECT * FROM prescription WHERE id = prescriptionId
    
    // 1. 检查处方状态
    IF prescription.status = 'CANCELLED' THEN
        RETURN Error("处方已作废，不可调配")
    IF prescription.status = 'EXPIRED' THEN
        RETURN Error("处方已过期，不可调配")
    IF prescription.status = 'DISPENSED' THEN
        RETURN Error("处方已发药，不可重复调配")

        validDays = 3
    ELSE

// 伪代码
FUNCTION selectBatchByFIFO(drugId, quantity, warehouseType):
    // 查询可用库存批次，按效期升序排列
    batches = SELECT * FROM drug_stock
              WHERE drug_id = drugId
              AND warehouse_type = warehouseType
              AND status IN ('NORMAL', 'NEAR_EXPIRY')
              AND quantity > 0
              ORDER BY expire_date ASC, create_time ASC
    
    selectedBatches = []

        selectedBatches.add({
            batchNo: batch.batch_no,
            expireDate: batch.expire_date,
            quantity: allocateQty
        })
        remainingQuantity = remainingQuantity - allocateQty
    
    IF remainingQuantity > 0 THEN
        RETURN Error("可用库存不足")

    safetyStock = drug.safety_stock         // 安全库存
    currentStock = getCurrentStock(drugId)  // 当前库存
    leadTime = drug.lead_time || 7          // 采购周期（天）

        currentStock: currentStock,

              WHERE drug_id = drugId
              AND batch_no = batchNo
              ORDER BY create_time ASC

FUNCTION determineOverallRiskLevel(cdsResults):
    // 风险等级定义
    // HIGH: 红色，强制退回
    // MEDIUM: 橙色，需确认处理
    // LOW: 黄色，提示注意
    // NONE: 绿色，通过

    riskLevels = [interactionRisk, allergyRisk, dosageRisk, incompatibilityRisk]

        action = 'FORCE_RETURN'  // 强制退回

        message = 'CDS校验通过'

        highCount: riskLevels.filter(r -> r = 'HIGH').length,
        mediumCount: riskLevels.filter(r -> r = 'MEDIUM').length,
        lowCount: riskLevels.filter(r -> r = 'LOW').length,

        message: message

    IF items IS EMPTY THEN RETURN 'NONE'
    riskOrder = ['HIGH', 'MEDIUM', 'LOW', 'NONE']
    maxRisk = 'NONE'
    FOR item IN items:
        IF riskOrder.indexOf(item.riskLevel) < riskOrder.indexOf(maxRisk) THEN
            maxRisk = item.riskLevel

#### BR-PHARM-031: DDD值统计计算规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PHARM-031 |
| 规则名称 | DDD值统计计算规则 |
| 规则类型 | 计算规则 |
| 适用对象 | 抗菌药物统计(antibiotic_statistics) |
| 规则描述 | 计算抗菌药物限定日剂量(DDD)使用强度 |
| 触发时机 | 定期统计时 |
| 校验逻辑 | ```
// 伪代码
FUNCTION calculateDDDStats(deptId, startDate, endDate):
    // DDD (Defined Daily Dose): 限定日剂量
    // DDDs = 用药总量 / DDD值
    // AUD (Antibiotic Use Density): 抗菌药物使用强度
    // AUD = DDDs / (同期出院人数 × 平均住院天数) × 100
    
    // 获取抗菌药物使用记录
    usage = SELECT d.drug_id, d.drug_name, d.ddd_value,
                   SUM(i.quantity) as total_quantity, d.unit
            FROM prescription p
            JOIN prescription_item i ON p.id = i.prescription_id
            JOIN drug d ON i.drug_id = d.id
            WHERE p.create_time BETWEEN startDate AND endDate
            AND d.antibiotic_level IS NOT NULL
            AND (deptId IS NULL OR p.dept_id = deptId)
            GROUP BY d.drug_id, d.drug_name, d.ddd_value, d.unit
    
    totalDDDs = 0
    drugStats = []
    
    FOR record IN usage:
        // 转换为DDD单位
        quantityInDDDUnit = convertUnit(record.total_quantity, record.unit, record.ddd_unit)
        // 计算DDDs
        ddds = quantityInDDDUnit / record.ddd_value
        
        drugStats.add({
            drugId: record.drug_id,
            drugName: record.drug_name,
            totalQuantity: record.total_quantity,
            dddValue: record.ddd_value,
            ddds: ddds
        })
        
        totalDDDs = totalDDDs + ddds
    
    // 获取出院患者信息
    dischargeInfo = SELECT COUNT(*) as patient_count,
                           AVG(DATEDIFF(discharge_time, admission_time)) as avg_stay_days
                    FROM inpatient_visit
                    WHERE discharge_time BETWEEN startDate AND endDate
                    AND (deptId IS NULL OR dept_id = deptId)
    
    // 计算AUD
    IF dischargeInfo.patient_count > 0 AND dischargeInfo.avg_stay_days > 0 THEN
        aud = totalDDDs / (dischargeInfo.patient_count * dischargeInfo.avg_stay_days) * 100
    ELSE
        aud = 0
    
    result = {
        deptId: deptId,
        startDate: startDate,
        endDate: endDate,
        totalDDDs: totalDDDs,
        dischargeCount: dischargeInfo.patient_count,
        avgStayDays: dischargeInfo.avg_stay_days,
        aud: aud,
        drugStats: drugStats
    }
    
    RETURN result

// 数据权限过滤
FUNCTION applyDataPermission(userId, query, permissionType):
    // 获取用户数据权限
    dataPermissions = SELECT * FROM user_data_permission
                      WHERE user_id = userId
                      AND permission_type = permissionType
    
    IF dataPermissions.length = 0 THEN
        // 无数据权限限制，返回全部数据
        RETURN query
    
    // 根据权限类型应用过滤
    SWITCH permissionType:
        CASE 'WAREHOUSE':
            warehouseIds = dataPermissions.map(p => p.warehouse_id)
            query = query + " AND warehouse_id IN (" + warehouseIds.join(",") + ")"
        
        CASE 'DRUG_TYPE':
            drugTypes = dataPermissions.map(p => p.drug_type)
            query = query + " AND drug_type IN ('" + drugTypes.join("','") + "')"
        
        CASE 'DEPARTMENT':
            deptIds = dataPermissions.map(p => p.dept_id)
            query = query + " AND dept_id IN (" + deptIds.join(",") + ")"
    
    RETURN query

    // 应用查询条件
    IF queryParams.drugId THEN
        query = query + " AND drug_id = " + queryParams.drugId
    
    // 执行查询

    stocks = SELECT * FROM drug_stock
             WHERE status IN ('NORMAL', 'NEAR_EXPIRY')

// 入库时效期检查
FUNCTION checkInboundExpiry(expireDate):
    warningDays = getConfig('expiry_warning_days', 90)
    minExpiryMonths = getConfig('min_expiry_months', 6)
    
    daysToExpiry = DATEDIFF(expireDate, NOW())
    monthsToExpiry = daysToExpiry / 30
    
    result = {
        daysToExpiry: daysToExpiry,
        monthsToExpiry: monthsToExpiry,
        status: 'NORMAL',
        warning: null
    }
    
    // 效期不足预警
    IF monthsToExpiry < minExpiryMonths THEN
        result.warning = "药品效期不足" + minExpiryMonths + "个月，是否继续入库？"
        result.needConfirm = true
    
    // 近效期判定
    IF daysToExpiry <= warningDays THEN
        result.status = 'NEAR_EXPIRY'
    
    // 已过期
    IF daysToExpiry <= 0 THEN
        result.status = 'EXPIRED'
        result.error = "药品已过期，禁止入库"
    
    RETURN result

FUNCTION dailyNarcoticAccountCheck():

    checkDate = DATE(checkTime)
    
    // 获取所有麻醉药品库存
    narcoticStocks = SELECT drug_id, batch_no, SUM(quantity) as total_quantity
                    FROM drug_stock
                    WHERE drug_type IN ('NARCOTIC', 'SPIRIT_I')
                    GROUP BY drug_id, batch_no

                batch_no: stock.batch_no,
                account_balance: 0,

                difference: stock.total_quantity,

                batch_no: stock.batch_no,

                actual_stock: stock.total_quantity,
                difference: difference,
                status: difference = 0 ? 'MATCHED' : 'MISMATCH'

        // 记录核对结果

                    NOW(), '资质' + qual.name + '将于' + daysToExpire + '天后到期'
                )
                
                // 发送预警
                sendAlert("供应商" + supplier.name + "资质" + qual.name + 
                         "将于" + daysToExpire + "天后到期（" + qual.expire_date + "），" +
                         "请及时联系供应商更新资质")
    
    RETURN Success

// 采购时资质检查
FUNCTION checkSupplierQualificationForPurchase(supplierId):
    supplier = SELECT * FROM supplier WHERE id = supplierId
    
    IF supplier.status = 'QUALIFICATION_EXPIRED' THEN
        RETURN Error("供应商资质已过期，禁止采购")
    
    qualifications = SELECT * FROM supplier_qualification
                    WHERE supplier_id = supplierId
    
    // 检查是否有有效资质
    validQuals = qualifications.filter(q => q.expire_date >= NOW())
    
    IF validQuals.length = 0 THEN
        RETURN Error("供应商无有效资质证书，禁止采购")
    
    RETURN Success
``` |
| 失败处理 | 过期供应商禁止采购 |
| 优先级 | P1 |
